# Claude Limit Meter

[Скачать VSIX v0.5.0](https://github.com/cleverplant/claude-limit-meter/releases/download/v0.5.0/claude-limit-meter-0.5.0.vsix) · [Описание релиза](https://github.com/cleverplant/claude-limit-meter/releases/tag/v0.5.0) · [English version](README.md)

![Панель Claude Quota](https://raw.githubusercontent.com/cleverplant/claude-limit-meter/main/claude-limit-web.png)

> **Важно про источник данных.** Расширение читает всё из **CLI-версии
> Claude Code** (терминальной) — из OAuth-кредов в
> `~/.claude/.credentials.json`, а для блока `/context` запускает свежую
> CLI-под-сессию в выбранной папке проекта. Оно **не** общается с
> графической панелью «Claude Code for VS Code» и не имеет доступа к её
> состоянию в памяти. Поэтому блок `/context` внутри web-панели Quota
> может показывать другую модель или другую разбивку токенов, чем GUI-
> панель Claude Code для того же проекта: CLI запускает новую короткую
> сессию, чтобы выполнить `/context`, а GUI показывает живую сессию,
> которая у него открыта. А вот цифры квоты 5ч / 7 дней приходят с
> аккаунта Anthropic и всегда совпадают между CLI и GUI.

## Что нового в 0.5.0

Это релиз с ломающими изменениями. Поверхность расширения сокращена до
одной части, которая надёжно работает во всех проверенных окружениях
Claude Code:

- Элемент статус-бара `cc-ctx` (процент + цветная полоска) **удалён**.
  Claude Code уже показывает ту же цифру в своём подвале; дубликат
  добавлял шум и поддержку — формула менялась каждый раз, когда
  Anthropic выпускала новую модель.
- PostCompact-хук авто-handoff **удалён**. Его канал вывода
  `systemMessage` тихо отбрасывается VS Code-расширением Claude Code
  (баг на стороне платформы), поэтому хук срабатывал, но его вывод
  никогда не доходил до чата. Держать его на диске означало обещать
  фичу, которая для VS Code-пользователей не работала.
- Кружок `🟢 / ⚪` в статус-баре **удалён** вместе с хуком, который он
  переключал.
- Слэш-команды `/kick-on` и `/kick-off` **удалены**. Без хука
  переключать нечего.
- Все настройки `claudeLimitMeter.*`, которые управляли этим, убраны:
  `updateIntervalSeconds`, `barLength`, `showBar`, `warnPercent`,
  `highPercent`, `criticalPercent`, `contextWindowOverride`,
  `show5hUsage`, `showWeeklyUsage`, `scanWindowHours`, `showKickStatus`.

Что **остаётся** и что нового для существующих пользователей:

- Элемент статус-бара `limit` и его web-панель Claude Quota остаются
  без изменений.
- Слэш-команда `/kick` остаётся. В отличие от PostCompact-хука, путь
  через слэш-команду идёт по обычному pipeline ассистент-сообщений,
  поэтому блок handoff всегда рендерится в чате с кнопкой Copy. Тело
  команды теперь на английском.
- При первой активации расширение автоматически устанавливает файл
  `~/.claude/commands/kick.md`, и `/kick` сразу доступен в любом чате
  Claude Code.
- При каждой активации расширение чистит остатки v0.4.x в
  `~/.claude/`: `scripts/kick-hook.js`, `commands/kick-on.md`,
  `commands/kick-off.md`, `.kick-installed-version`, `.kick-disabled`,
  `.kick-log` и запись `hooks.PostCompact`, у которой `command`
  содержит `kick-hook.js`. Остальные ключи `settings.json` не
  трогаются.

## Для пользователя

Что расширение добавляет в VS Code и как этим пользоваться каждый день.

### Один элемент в статус-баре: `limit`

В правом нижнем углу VS Code появляется один новый элемент:

```
$(pulse) limit
```

Клик открывает web-панель Claude Quota.

### Панель Quota (по клику `limit`)

Панель открывается во вкладке редактора и состоит из двух блоков:

- **Квота Anthropic** — скользящий 5-часовой блок и 7-дневный расход в
  процентах, время до сброса каждого окна и значок тарифа
  (`Pro` / `Max` / `Free`). Цифры приходят с серверов Anthropic через
  тот же эндпоинт, что использует официальное приложение Claude Code.
  Кешируются локально на 5 минут — повторный клик на `limit` возвращает
  кешированные данные; кнопка `↺ Refresh` принудительно перезапрашивает.
- **Контекст проекта (`/context`)** — выберите папку рабочей области
  из выпадающего списка и нажмите `/context`. Расширение запускает CLI
  Claude Code в этой директории и показывает разобранный ответ: имя
  модели, общий расход контекста и разбивку по категориям (System
  prompt, Memory files, Skills, Messages и т. д.).

> ⚠️ Блок `/context` запускает CLI-под-сессию, а **не** вашу живую
> сессию Claude Code в VS Code. Модель и проценты могут отличаться от
> того, что показывает GUI-панель — см. предупреждение в самом верху.

### Слэш-команда `/kick`

`/kick` — это слэш-команда, которая создаёт готовый к копированию блок
handoff, чтобы перейти из заполненного чата в свежий без потери
ориентации.

**Что она делает.** Когда вы пишете `/kick` в любом чате Claude Code,
команда собирает git-состояние проекта, читает `plans/plan.md` (если
он есть), кратко описывает, что было сделано в текущей сессии, и
печатает в чат fenced-блок с кнопкой Copy. Если перед `/kick` вы
запустили `/compact`, команда дополнительно вытащит post-compact
саммари из JSONL сессии и встроит его «как есть» во второй блок.

**Почему она работает там, где авто-хук не работал.** `/kick` — это
обычная слэш-команда. Её вывод идёт через тот же pipeline
ассистент-сообщений, что и любой другой ответ, поэтому он одинаково
рендерится во всех поверхностях Claude Code (CLI, VS Code-расширение,
JetBrains-плагин). PostCompact-хук в v0.4.x пытался делать то же самое
автоматически после compact, но его канал `systemMessage` тихо
отбрасывается VS Code-расширением Claude Code. v0.5.0 убирает
авто-хук полностью и оставляет только ручную команду, которая
работает.

**Рекомендуемый порядок, когда контекст уже близок к лимиту:** сначала
`/compact`, потом `/kick`. Compact пишет в JSONL структурированное
саммари, которое `/kick` встраивает в свой вывод.

### Установка

Из папки проекта в PowerShell:

```powershell
./install.ps1
```

Затем в VS Code — **`Developer: Reload Window`**. При первой активации
команда `/kick` сама установится в `~/.claude/commands/kick.md`, а
остатки v0.4.x-хука будут удалены.

### Удаление

Удаление `.vsix` **не** удаляет автоматически
`~/.claude/commands/kick.md` — VS Code не вызывает `deactivate()` при
удалении расширения, поэтому слэш-команда останется доступной. Если
это то, что вам нужно (более чистый список расширений, `/kick` всё ещё
работает), можно на этом остановиться.

Для полной чистки:

1. `Extensions → Claude Limit Meter → Uninstall`.
2. Удалите `~/.claude/commands/kick.md` (опционально — слэш-команда
   перестанет работать).
3. `Developer: Reload Window`.

**Удаление через агента.** Готовый промпт для Codex / Claude Code
лежит в [prompts/uninstall-full.md](prompts/uninstall-full.md).

## Техническая часть

Как это устроено внутри и где живут данные.

### Источники данных

Три независимых входа:

| Блок | Источник | API-вызов? |
|---|---|---|
| Квота 5ч / 7д | `GET https://api.anthropic.com/api/oauth/usage` с OAuth-токеном из `~/.claude/.credentials.json` | Да (Anthropic) |
| Значок тарифа (`Pro` / `Max`) | `GET https://api.anthropic.com/api/oauth/profile` (тот же токен) | Да (Anthropic) |
| Блок `/context` | Запускает `C:\Tools\claude-wrap.exe` в выбранной папке проекта, шлёт `/context` в stdin, парсит markdown-ответ | Косвенно — через CLI |

Расширение не отправляет содержимое чатов наружу, не читает тела
сообщений и не требует от вас вводить API-ключ. OAuth-токен — это тот
же токен, который уже использует CLI.

### Все настройки

```text
claudeLimitMeter.usagePageUrl       # URL по команде Open Usage Page
claudeLimitMeter.statusBarPriority  # позиция элемента limit (больше = левее)
claudeLimitMeter.textColor          # цвет текста в статус-баре
```

### Команды Command Palette

```text
Claude Limit Meter: Open Usage Page (claude.ai)
Claude Limit Meter: Show Quota Limits
```

### Файлы, которые записываются в `~/.claude/`

При каждой активации (идемпотентно):

```text
~/.claude/commands/kick.md          — тело слэш-команды /kick
```

Ничего больше не пишется. Прежний след v0.4.x
(`scripts/kick-hook.js`, `commands/kick-on.md`, `commands/kick-off.md`,
`.kick-installed-version`, `.kick-disabled`, `.kick-log` и запись
`hooks.PostCompact` внутри `settings.json`) удаляется автоматически
при первой активации v0.5.0 на машине, где раньше был установлен
v0.4.x.

### Почему лимиты — реальная квота, а не подсчёт токенов локально

Ранние версии пытались агрегировать расход токенов локально по JSONL,
чтобы оценить блоки 5ч и неделю. Эта цифра всегда была неточной:
Claude Code не сохраняет в логах rate-limit-заголовки Anthropic, а
сами лимиты тарифа сильно отличаются между Pro / Max 5× / Max 20× /
API. Начиная с v0.4.0 панель использует тот же эндпоинт, что и
официальное приложение Claude Code — `/api/oauth/usage` — и показывает
реальные проценты и время сброса. 5-минутный кеш страхует от лишних
запросов.
